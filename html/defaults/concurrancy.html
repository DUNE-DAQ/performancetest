Concurrency will be maintained as a single point of test responsibility: A single RU will be tested against the target load test criteria. Any parallel resource usage not responsible for telemetry and profiling purposes on the RU will invalidate the test and its results.

In order to verify this criteria, needed items to be logged are:

  • Active user sessions on the RU during the load test
  • Active processes and services running on the RU during the load test

Examples:

  • Control processes running on the RU
  • DAQ Services on the RU
  • Parallel emulator tests are executed on the RU
